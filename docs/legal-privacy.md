> 한국 개인정보보호법(PIPA)·형법·통신/금융 규제 관점의 리스크 정리와 개인정보 최소화 설계.
> 출처: `roadmap.md` §7.1~7.2 를 실사용 가능한 정책으로 확장. 짝 문서: [`expression-guidelines.md`](expression-guidelines.md), [`abuse-defense.md`](abuse-defense.md), [`dpia-checklist.md`](dpia-checklist.md).

> ⚠️ **면책 / 필수 조건**
> 이 문서는 엔지니어링 팀의 **내부 설계 기준**이며 법률 자문이 아니다.
> **출시 전 반드시 한국 변호사·개인정보 전문가의 실제 법률검토를 받아야 한다**(형법 명예훼손·업무방해, PIPA, 정보통신망법, 신용정보법, 통신비밀보호법, 전기통신사업법, 위치정보법). 검토 전까지 실서비스 공개 배포는 금지한다.

## 1. 관련 법령 개요

| 법령 | 관련 조항(요지) | ScamGraph 접점 |
|---|---|---|
| **개인정보보호법(PIPA)** | 수집·이용 최소화, 목적 제한, 동의/법적근거, 정보주체 권리, 안전조치, 국외이전, 영향평가 | 전화·계좌·IP·검사기록·신고 처리 전반 |
| **형법 §307 명예훼손 / §314 업무방해** | 사실/허위 적시로 명예 훼손, 위계·위력 업무방해 | 특정 번호·계좌·도메인·사업자를 "사기"로 표기할 때 |
| **정보통신망법** | 온라인 명예훼손(§70), 개인정보 관련 특례, 위치정보 | 공개 게시·알림·확장 배너 |
| **신용정보법** | 신용정보(계좌 등) 수집·이용·제공 제한 | 계좌 관련 정보 처리 |
| **통신비밀보호법** | 통신 내용 감청·수집 금지 | SMS 원문·통화 내용 처리(→ 원칙적 미수집) |
| **전기통신사업법** | 발신번호 변작 금지, 스팸 규제 | 전화 신호 처리·표현 |
| **위치정보법** | 위치정보 수집·이용 동의 | 정확위치 미수집(§2), GeoIP 국가 단위만 |

> 국내 사기·피싱 정보 공유는 공익적 성격이 강하지만, **공익 목적이라도 개별 식별자를 단정하면 명예훼손이 성립할 수 있다.** 표현 규칙([`expression-guidelines.md`](expression-guidelines.md))이 1차 방어선이고, 이 문서의 최소화·마스킹이 2차 방어선이다.

| R3 | 부정확 정보 확산 | PIPA(정확성), 민사 손배 | 오탐이 검색·공유로 확산 | `noindex`, 자동 마스킹, tombstone, 오탐 신고 채널 |

| R5 | 통신 내용 침해 | 통신비밀보호법 | SMS 원문·통화 녹음 서버 전송 | 원문 기기 밖 미전송, URL/indicator 만 |
| R6 | 신용정보 오남용 | 신용정보법 | 계좌정보 재배포 | 계좌 keyed hash/마스킹, 제공 제한 |
| R7 | 국외이전 위반 | PIPA §28-8 | 외부 SaaS(위협 feed·클라우드)로 개인정보 이전 | 이전 고지·최소화·hash lookup(§5) |
| R8 | 정보주체 권리 미보장 | PIPA §35~39 | 열람·삭제·정정 요청 미처리 | 삭제/이전/정정 절차(§6), 이의제기(abuse-defense §4) |

## 3. 개인정보 최소화 원칙 (핵심 설계)

> "**수집하지 않은 데이터는 유출·오남용될 수 없다.**" ScamGraph 는 판정에 필요한 **indicator(신호)만** 남기고 원문·식별 정보는 최대한 수집하지 않는다.

### 3.1 데이터 항목별 처리 정책

| 데이터 | 수집 여부 | 처리 방식 | 근거 |
|---|---|---|---|
| **SMS/문자 원문** | ❌ 서버 미전송 | 기기 내 로컬 규칙엔진(`quick_assess` Dart 포팅)에서 처리 → **URL/도메인만** 추출·전송 | roadmap §4.1, §7.2 |
| **통화 내용/녹음** | ❌ 미수집 | 번호·행동 신호만. 녹음/분석 없음 | roadmap §4.2 |
| **전화번호** | △ 조회는 hash | 서버 조회는 **HMAC/keyed-hash lookup**(원문 번호 서버 저장 최소화) | roadmap §7.2 |
| **계좌번호** | △ 마스킹/hash | **마스킹 + keyed hash**, 명의자 정보 **미수집** | roadmap §1.4, §7.2 |
| **계좌 명의자** | ❌ 미수집 | 기본 미수집 | roadmap §1.4 |
| **IP 주소** | △ 최소 | 판정·SSRF 로그 목적 최소 보관, 공개는 `/24`·국가 단위 | §2.5(expression) |
| **정확 위치(GPS)** | ❌ 미수집 | 지도는 GeoIP **국가 집계**만, 오차 표기 | roadmap §3.3, 위치정보법 |
| **기기 ID / 광고 ID** | ❌ 미사용 | 광고ID 미사용, device ID 최소 | roadmap §7.2 |
| **검사 기록(history)** | △ opt-in | **명시적 opt-in** 시에만 저장, 기본 미저장 | roadmap §7.2 |
| **방문 기록(브라우징)** | ❌ 미전송 | 확장은 로컬 blocklist 로 판정, 일반 URL 서버 미전송 | roadmap §5.1 |
| **신고자 신원** | △ 최소·비공개 | 신고 처리·reputation 목적 최소, **어디에도 미공개** | expression §5 |

### 3.2 처리 흐름 & 안전조치

- **목적별 동의 분리** — 검사, 검사기록 저장, 알림/구독, 가족 보호 모드는 **각각 별도 동의**.
- **로그 redaction** — 서버 로그·trace 에서 전화/계좌/원문을 마스킹하여 기록.
- **암호화** — 저장 시(at rest) 및 전송 시(in transit) 암호화. 모바일은 `EncryptedSharedPreferences`/Keystore(roadmap §4.10).
- **접근통제** — 전체 식별자·상세 그래프는 인증된 조사자만. 관리자 MFA·RBAC/ABAC(roadmap §6 보안).
- **최소 보존** — §4 보존기간 경과 시 파기/익명화. soft delete + tombstone.

## 4. 보존기간 정책

| 데이터 | 보존기간(제안) | 파기/전환 |
|---|---|---|
| 검사 기록(opt-in) | 사용자 지정 또는 기본 90일 | 만료 시 파기, 사용자 즉시 삭제 가능 |
| 스캔 판정(`scans`) | 판정 근거용 최소 기간(예 180일) | 개인식별 요소 제거 후 통계만 유지 |
| 신고(`reports`) | 무결성·이의제기 대응 기간 | 신고자 신원은 분리 보관·조기 파기 |
| IP/보안 로그 | 침해대응 목적 최소(예 30~90일) | 마스킹 후 통계화 |
| blocklist / 판정(공개) | 유효기간 + 만료(feed manifest) | 만료·정정 시 tombstone·delta 회수 |
| audit log(조사자 조회) | 감사 목적 장기 | 접근 통제·무결성 보장 |

> 구체 기간은 **법률검토에서 확정**한다(위 값은 설계 기본값·placeholder).

## 5. 국외이전 · 외부 SaaS 고지

ScamGraph 는 외부 위협 feed·클라우드·검사 소스를 사용한다. 이 중 **개인정보가 국외 사업자로 이전되는 경로**를 식별·고지·최소화한다(PIPA §28-8).

| 외부 서비스(예) | 목적 | 이전 항목 | 국가 | 최소화 조치 |
|---|---|---|---|---|
| Google Safe Browsing / Web Risk | URL 평판 | **URL hash(프리픽스)** — 원문 URL 미전송 | 미국 | hash lookup 옵션(roadmap §2.2) |
| VirusTotal / urlscan | URL·파일 평판 | URL·해시(사용자 동의·deep scan 시만) | 미국 등 | opt-in, 원문 최소 |
| OpenPhish/URLhaus/ThreatFox/PhishTank | 위협 feed 수신 | (수신 위주, 개인정보 이전 적음) | 해외 | 재배포·라이선스 registry 관리(roadmap §1.7) |
| 클라우드 인프라 | 호스팅 | 서비스 데이터 | 배포 리전에 따름 | 리전 선택·암호화·egress 통제 |

**규칙**
- 개인정보가 포함될 수 있는 조회는 **hash lookup 우선**(원문 전송 회피).
- 각 소스의 **라이선스·호출한도·재배포 가능여부**를 registry 로 관리(roadmap §1.7).
- 국외이전이 발생하는 항목은 **개인정보 처리방침에 이전 국가·항목·목적·거부권**을 고지.
- deep scan 등 외부 전송이 필요한 동작은 **사용자가 명시적으로 실행한 경우에만**(자동 서버 전송 금지, roadmap §5.6).

## 6. 정보주체 권리 보장

PIPA §35~39 에 따른 권리를 절차로 보장한다. (판정 이의제기는 [`abuse-defense.md`](abuse-defense.md) §4 참조.)

| 권리 | 처리 방식 | 접점 |
|---|---|---|
| 열람 | 본인 검사기록·저장 항목 조회 | 계정 설정 |
| 정정 | 오탐/오류 정정 요청 | 이의제기 form(누구나 접근) |
| 삭제 | 검사기록·저장 항목 삭제(즉시) | 계정 설정 · 요청 채널 |
| 처리정지 | 알림/구독/저장 중단 | 목적별 동의 철회 |
| **이전(export)** | 본인 데이터 내보내기 | 삭제/export 요청 채널(roadmap §7.2) |

- 요청 처리 **SLA** 를 정의하고 로그로 추적(투명성 보고서에 통계 반영, [`transparency-report-template.md`](transparency-report-template.md)).
- 신고자 개인정보는 정보주체 권리 처리 과정에서도 **미공개**.

## 7. 출시 전 법률검토 체크리스트

- [ ] 개인정보 처리방침(수집항목·목적·보존·제3자·국외이전·권리) 작성 및 변호사 검토
- [ ] 표현 가이드([`expression-guidelines.md`](expression-guidelines.md)) 준수 여부 법적 확인(명예훼손/업무방해)
- [ ] DPIA 완료([`dpia-checklist.md`](dpia-checklist.md))
- [ ] 계좌/신용정보 처리의 신용정보법 적법성 검토
- [ ] SMS/통화 처리의 통신비밀보호법 저촉 여부 확인(원문 미수집 설계 검증)
- [ ] 발신번호·전화 신호 처리의 전기통신사업법 검토
- [ ] 국외이전 항목·경로 목록화 및 고지문 확정
- [ ] 보존기간(§4) 법적 확정
- [ ] 미성년/피보호자(가족 모드) 동의 구조 검토(스토킹 악용 방지 포함, roadmap §4.8)
- [ ] 오탐·이의제기·손해배상 리스크에 대한 면책·대응 절차 확정
- [ ] 데이터 침해 시 신고·통지 절차(PIPA) 수립

> 이 체크리스트가 모두 통과되기 전에는 **공개 실서비스 배포 금지.** 해커톤 데모는 시드/폴백 데이터 기준으로 운영하며 실제 개인 식별자 노출을 피한다.
