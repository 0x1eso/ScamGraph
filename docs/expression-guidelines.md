# ScamGraph 표현 가이드 (Expression Guidelines)

> 판정 결과를 화면·API·PDF·알림·확장·모바일 어디에 노출하든 **동일하게 적용되는 표현 규칙**.
> 출처: `roadmap.md` §7.1 을 실사용 가능한 규정으로 확장. 짝 문서: [`legal-privacy.md`](legal-privacy.md), [`abuse-defense.md`](abuse-defense.md).
> 상태: 초안 — **출시 전 법률검토 필요**(§7 참조).

---

## 0. 왜 이 문서가 필요한가

ScamGraph 는 전화번호·계좌·도메인·IP 를 위험도와 함께 공개한다. 그런데 이 식별자들의 **소유자가 곧 범죄자는 아니다.**

- **발신번호(전화)** — 발신번호 표시는 조작(spoofing) 가능. 표시된 번호의 실제 명의자는 피해자일 수 있다.
- **계좌** — 대포통장은 명의도용·강요로 개설된 경우가 많다. 명의자 ≠ 사기 수익자.
- **도메인/IP** — 정상 사이트가 해킹(compromise)당해 피싱 페이지가 얹히거나, 공용 호스팅(shared hosting)·클라우드 하위도메인이 악용된다. 등록자 ≠ 악용자.

따라서 "이 번호는 사기범", "이 계좌 주인은 범죄자" 같은 **단정 표현은 명예훼손·부정확정보·개인정보 침해 리스크**를 만든다. 이 문서는 그 리스크를 표현 단계에서 차단한다.

## 1. 3대 표현 원칙

| 원칙 | 내용 |
|---|---|
| **① 단정 금지** | 식별자를 "사기범/범죄자/사기 계좌"로 **단정하지 않는다.** 항상 "관측/신고/정황" 프레임으로 서술한다. |
| **② 사실·추정·신고 분리** | 확인된 사실(feed·기술신호)과 추정(그래프 상관), 사용자 신고를 **시각적으로 분리**해 표기한다(§3). |
| **③ 근거·출처·시각 동반** | 모든 판정에는 `reasons`(근거), 출처, 관측시각, 불확실성을 함께 노출한다. 점수만 단독 노출 금지. |

## 2. 승인된 표현 매핑 표

**금지 표현 → 승인 표현.** UI 카피·PDF·알림 문구·API `label` 은 아래 우측 열만 사용한다.

### 2.1 전화번호

| 금지 (단정) | 승인 (관측·정황) |
|---|---|
| "이 번호는 사기범입니다" | "**사기 신고에 사용된 것으로 관측된** 발신번호" |
| "보이스피싱 범인 번호" | "기관 사칭 신고가 접수된 발신번호 (발신번호는 **조작 가능**)" |
| "이 사람이 사기꾼" | "표시 발신번호이며, **실제 명의자는 피해자일 수 있음**" |

> 전화 관련 화면에는 **항상** "발신번호는 조작될 수 있으며, 표시된 번호의 명의자가 범죄자라는 의미가 아닙니다" 경고를 병기한다.

### 2.2 계좌번호

| 금지 (단정) | 승인 (관측·정황) |
|---|---|
| "사기 계좌", "이 계좌 주인은 사기범" | "**신고 및 연관 정황이 있는** 계좌" |
| "대포통장 주인" | "송금 피해 신고가 접수된 계좌 (명의자 ≠ 수익자일 수 있음)" |
| "이 계좌로 보내면 사기" | "**주의가 권고되는** 계좌 — 송금 전 확인 필요" |

> 계좌 명의자 정보는 **수집·표시하지 않는다**(기본). 계좌는 마스킹 또는 keyed hash 로만 노출한다(§5).

### 2.3 도메인 / URL / IP

| 금지 (단정) | 승인 (관측·정황) |
|---|---|
| "이 사이트 운영자는 사기범" | "피싱에 사용된 것으로 **관측된** 도메인" |
| "이 IP는 범죄 서버" | "다수 위험 도메인이 **함께 호스팅된** IP (공용 호스팅 가능성 있음)" |
| "이 도메인 = 사기 조직" | "**공통 인프라를 공유하는 캠페인 후보** (근거 N개)" |
| "해킹당한 사이트 = 악성" | "정상 사이트가 **침해되어(compromise)** 악용된 것으로 추정 — 루트 도메인 전체를 악성으로 단정하지 않음" |

### 2.4 사람 / 조직 / 캠페인

| 금지 | 승인 |
|---|---|
| "동일 사기 조직" | "**동일 캠페인 84%** 추정 (공유 근거 4개)" |
| 실명·업체명 특정 후 "사기범" | 실명/업체 **미표기**. 캠페인은 자동 생성 코드명(예: `CAMP-2026-041`)으로 지칭 |

### 2.5 판정 등급(grade)별 문구

엔진 `grade` (`safe|caution|warning|danger`) 를 사용자 문구로 옮길 때:

| grade | 간편 모드 문구 | 분석 모드 문구 | 금지 |
|---|---|---|---|
| `safe` | "현재까지 위험 근거가 확인되지 않음" | "안전(근거 충분) — 장기 유지·안정 인프라" | "100% 안전", "안전 보장" |
| `caution` | "주의 — 확인 후 이용" | "주의 신호 일부 관측, 악성 미확정" | "사기 의심 확정" |
| `warning` | "경고 — 민감정보 입력 주의" | "다수 위험 신호 관측, 확정 근거 대기" | "사기 사이트" |
| `danger` | "열지 마세요 · 송금 마세요" | "확인된 위협 (feed/기술신호 근거)" | "범죄자 사이트", "이 사람이 사기범" |

> **중요:** `safe` 는 "안전 보장"이 아니라 **"아직 악성 미확인"**이다. 판정 등급을 **"안전(근거충분)" vs "아직 악성 미확인"** 두 축으로 분리해 표기한다(roadmap §1.5).

## 3. 사실 / 추정 / 사용자 신고 분리 원칙

모든 근거(`reasons[]`)와 그래프 엣지는 **증거 등급(evidence tier)** 을 달고 노출한다.

| 등급 | 정의 | 출처 예 | UI 표기 |
|---|---|---|---|
| **사실(confirmed)** | 검증된 외부 feed 또는 재현 가능한 기술신호 | OpenPhish·URLhaus·ThreatFox·경찰청, TLS/DNS/WHOIS 관측, homoglyph diff | 실선 배지 · "확인됨" |
| **추정(inferred)** | 그래프 상관·유사도로 도출한 연결 | 공유 IP/인증서/GTM, 커뮤니티 탐지, centrality | 점선 배지 · "추정 84%" |
| **신고(reported)** | 시민·사용자 신고 | `POST /report` 사용자 신고 | 물음표 배지 · "N명 신고" |

**규칙**
- 세 등급은 **색·아이콘·라벨로 구분**한다(색만으로 구분 금지 — 색각이상 대응).
- 신고(reported) 만으로 `danger` 등급을 부여하지 않는다 → 승격 정책은 [`abuse-defense.md`](abuse-defense.md) §3.
- PDF/공유 이미지에도 이 등급 라벨을 유지한다.

## 4. 불확실성 · 출처 · 관측시각 표기 규칙

모든 판정 노출부에 **최소 4요소**를 함께 표기한다.

| 요소 | 형식 예 | 비고 |
|---|---|---|
| **불확실성** | "추정 84%", "신뢰구간 71–89%" | 신고는 Wilson score/Beta-Binomial 구간(roadmap §1.8) |
| **출처** | "출처: URLhaus + 시민신고 3건" | feed 명·신고 수 명시 |
| **관측시각** | "최초 관측 2026-06-30 · 최근 2026-07-10" | `first_seen`/`last_seen` |
| **규칙 버전** | "rules v2026.07 · 판정ID r-9f3a" | 재현 가능성 확보 |

**추가 규칙**
- **금전 피해액을 근거 없이 직접 추산하지 않는다.** 추산 시 계산식·가정을 반드시 공개(roadmap §2.7).
- 판정 변경 이력이 있으면 "quick→deep 재판정됨" 을 표기.
- 이의제기 진행 중인 항목은 **"이의제기 검토 중"** 배지를 표시(§7, abuse-defense §4).

## 5. 공개 화면 마스킹 규칙

인증되지 않은 공개 화면(익명 사용자·공유 이미지·검색결과)에는 식별자를 **마스킹**한다. 전체값은 **인증된 조사자 화면 / 수사용 case-file 전체본** 에서만 노출(roadmap §2.5, §7.5).

| 대상 | 공개(익명) 표시 | 인증 조사자 표시 |
|---|---|---|
| 전화번호 | `010-****-1234` (뒤 4자리) | 전체 |
| 계좌번호 | `국민 ***-**-**89` (은행+뒤 2자리) | 전체 |
| 도메인 | 전체 표기 가능하되 **클릭 비활성·`defang`**(`shinhan-otp[.]xyz`) | 활성 링크는 격리 뷰어에서만 |
| IP | `/24` 대역 또는 국가만 (`203.0.113.x`) | 전체 |
| 이메일 | `ab***@***.com` | 전체 |
| 신고자 | **항상 비공개** (익명·인증 화면 모두) | 비공개 (audit log 별도) |

**원칙**
- 결과 **공유(이미지/링크) 시 자동 마스킹** 적용(roadmap §3.4).
- URL 은 공개 화면에서 **defang**(`hxxp`, `[.]`) 하여 실수 클릭·재확산 방지.
- 마스킹 해제는 소유권 증명 또는 조사자 권한 + audit log 를 전제로 한다.

## 6. 검색엔진 색인 · 대량수집 제한 규칙

특정 개인/사업자가 "내 번호/계좌/도메인이 사기로 검색된다"는 2차 피해를 막기 위한 규칙.

| 항목 | 규칙 |
|---|---|
| **검색엔진 색인** | 식별자별 상세 결과 페이지는 `noindex, nofollow` · `robots.txt` 차단. 공개 가능한 것은 캠페인 단위 통계뿐. |
| **reverse lookup 방지** | "번호/계좌 → 소유자" 역조회 기능 제공 금지. lookup 은 HMAC/keyed-hash 기반 일치 확인만. |
| **대량수집 방지** | 익명 결과는 요약만, 세부 그래프·전체 식별자는 인증 조사자만. bulk/batch API 는 승인제 + rate limit(abuse-defense §5). |
| **permalink** | 공개 permalink 는 캠페인(코드명) 단위만. 개인 식별자 단위 permalink 금지. |
| **딥링크 스크래핑** | anomaly detection 으로 대량 조회 탐지 시 요약만 반환·속도 저하. |

## 7. 이의제기 · 정정 상태 표기

- 이의제기 접수 항목: **"이의제기 검토 중"** 배지 + (피해가 크면) 공개 표시 **임시 suppress**.
- 정정 완료 항목: 이전 판정에 **tombstone**("정정됨") 유지, 변경 사유 기록.
- 모바일/확장에는 **긴급 정정 delta** 로 회수 사실을 전파(roadmap §7.4).
- 상세 절차·SLA 는 [`abuse-defense.md`](abuse-defense.md) §4.

## 8. UI 카피 체크리스트 (배포 전)

- [ ] "사기범/범죄자/사기 계좌" 등 **단정 표현이 하나도 없는가?**
- [ ] 전화 화면에 "발신번호 조작 가능·명의자≠범죄자" 경고가 있는가?
- [ ] 계좌 명의자 정보가 노출되지 않는가?
- [ ] 침해 도메인을 루트 전체 악성으로 단정하지 않는가?
- [ ] 사실/추정/신고가 **색+아이콘+텍스트**로 구분되는가?
- [ ] 불확실성·출처·관측시각·규칙버전 4요소가 표기되는가?
- [ ] `safe` 를 "안전 보장"이 아니라 "미확인"으로 서술하는가?
- [ ] 공개 화면 마스킹·URL defang 이 적용되는가?
- [ ] 신고자 개인정보가 어디에도 노출되지 않는가?
- [ ] 금전 피해액을 가정 공개 없이 추산하지 않는가?
- [ ] 상세 페이지에 `noindex` 가 걸려 있는가?

## 9. 금지 표현 블랙리스트 (자동 검출 권장)

배포 파이프라인/린트에서 아래 문자열 사용을 경고한다.

```
사기범  범죄자  범인  사기꾼  사기 계좌  사기 번호
100% 안전  안전 보장  절대 안전  확실한 사기  틀림없이
이 사람  이 업체(가 사기)  대포통장 주인  범죄 서버
```

> 대안: "관측된 / 신고된 / 정황이 있는 / 후보 / 추정 / 주의 권고" 프레임으로 치환.

## 부록 A. API `label` 생성 규칙

엔진/게이트웨이가 `label` 을 만들 때:

- `Target` 노드: 마스킹·defang 후 도메인/식별자 문자열.
- `Phone`/`Account`: **마스킹된 값 + 관측 프레임 접두**(예: "신고 관측 · 010-\*\*\*\*-1234").
- `Campaign`: 코드명(`CAMP-YYYY-NNN`) — 실명/업체명 금지.
- grade→문구 변환은 §2.5 표를 단일 소스로 사용(중복 하드코딩 금지, 공용 유틸에서 매핑).
