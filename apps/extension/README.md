# ScamGraph 방패 — 브라우저 방화벽 (Chrome/Edge, Manifest V3)

방문 사이트를 사기·피싱 위협 목록과 대조해 **로컬에서 차단**하는 브라우저
방화벽입니다. 핵심 설계는 **개인정보 보존형 차단**입니다.

> ⚠️ **런타임 미검증 안내:** 이 환경에는 브라우저가 없어 실제 로딩·동작을
> 확인하지 못했습니다. 아래 코드는 MV3 규격에 맞춰 작성했고 모든 JSON/JS
> 문법과 핵심 로직(퓨니코드 디코더·해시 무결성)은 정적으로 검증했습니다.
> 최종 동작은 `chrome://extensions`에서 로드해 확인하세요.

---

## 🔒 개인정보 보존형 차단 (핵심 세일즈 포인트)

일반적인 안전 브라우징 확장은 **방문하는 모든 URL을 서버로 전송**해 판정을
받습니다. ScamGraph 방패는 그렇지 않습니다.

1. 백그라운드 서비스 워커가 **약 30분마다 한 번** 게이트웨이에서 위협 목록
   전량(`GET /api/blocklist/snapshot`)을 받아옵니다.
2. 응답의 `count`·`hash`를 **로컬에서 SHA-256으로 재계산해 무결성을 검증**한
   뒤, 도메인 항목을 `chrome.declarativeNetRequest`(DNR) **동적 리다이렉트
   규칙**으로 변환합니다.
3. 이후 모든 차단은 **브라우저 내부에서 로컬로** 처리됩니다. 즉,
   **어떤 URL을 방문하는지는 서버로 전송되지 않습니다.**
4. 차단된 페이지는 확장 내부의 `blocked.html` 인터스티셜로 리다이렉트됩니다.
   이때 원본 주소를 쿼리에 넣지 않고, **불투명한 인시던트 id(`?r=<n>`)** 만
   전달하며 실제 도메인은 `chrome.storage.session`에서 조회합니다.
5. 게이트웨이가 꺼져 있어도 **마지막으로 받은 목록으로 계속 보호**합니다
   (오프라인 세이프).

서버로 나가는 통신은 딱 두 가지뿐입니다.
- 주기적 위협 목록 다운로드(내 방문 기록과 무관)
- **사용자가 직접** 누른 "상세검사"·"신고" (명시적 행위)

---

## 설치 (압축해제된 확장 프로그램 로드)

1. Chrome/Edge 주소창에 `chrome://extensions` (Edge는 `edge://extensions`) 입력
2. 우측 상단 **개발자 모드** 켜기
3. **압축해제된 확장 프로그램 로드** 클릭
4. 이 저장소의 `apps/extension` 디렉터리 선택
5. 툴바에 방패 아이콘이 나타나면 완료

> 게이트웨이(`http://localhost:8080`)가 실행 중이면 위협 목록 동기화와
> 상세검사가 동작합니다. 꺼져 있어도 확장은 마지막 목록/로컬 휴리스틱으로
> 안전하게 동작합니다.

## 구성 요소

| 파일 | 역할 |
|---|---|
| `manifest.json` | MV3 매니페스트 (권한·DNR·사이드패널·컨텍스트메뉴) |
| `background.js` | 서비스 워커 — 목록 동기화, DNR 규칙 생성, 임시 허용, 배지/알림, 메시지 허브 |
| `lib/heuristics.js` | 공유 로컬 판정 로직 — 퓨니코드(IDN) 디코더, 도메인 매칭, 휴리스틱 (순수 함수, 네트워크 없음) |
| `content.js` | 링크 호버 위험 툴팁(Shadow DOM·디바운스 150ms) + 자격증명/OTP 폼 제출 가드 |
| `blocked.html` / `blocked.js` | 차단 인터스티셜 — 도메인·근거·안전하게 뒤로가기·오탐 신고·고급 계속 |
| `popup.html` / `popup.js` | 현재 탭 로컬 판정, 목록 버전/동기화, 오늘 차단 수, 상세검사, 신고 |
| `options.html` / `options.js` | 개인 허용목록, 보호 수준, 익명 통계 옵트인 |
| `sidepanel.html` / `sidepanel.js` | 현재 사이트/선택 대상의 전체 근거 상세 패널 |
| `ui.css` | 공유 디자인 토큰 (웹앱 `globals.css` 팔레트와 통일) |
| `icons/` | 방패 아이콘 (16/32/48/128) |

## 기능

- **접속 차단 (DNR):** 위협 목록 도메인 접속을 로컬 리다이렉트로 차단하고
  경고 인터스티셜을 표시. 규칙은 목록 동기화 때마다 재생성.
- **링크 호버 방패:** 링크에 마우스를 올리면 로컬 판정으로 위험 툴팁 표시.
  퓨니코드(IDN) 도메인은 사람이 읽을 수 있는 실제 표기로 디코딩해 보여줌
  (예: `xn--80ak6aa92e.com → аррӏе.com`).
- **자격증명/OTP 폼 가드:** 위험 사이트에서 비밀번호·인증번호 입력 폼을 제출
  하기 직전 경고. **입력 값은 절대 읽지 않고** 필드 종류/폼 action 호스트만 확인.
- **컨텍스트 메뉴:** 링크/선택 텍스트 우클릭 → "ScamGraph로 검사" → 사이드 패널.
- **사이드 패널:** 판정 등급·위험도·근거·귀속 조직·피드·커뮤니티 신고 상세.
- **팝업:** 현재 탭 즉시 판정 + 목록 버전/동기화 시각 + 오늘 차단 수 + 상세검사.
- **옵션:** 표준/엄격/모니터 보호 수준, 개인 허용목록, 익명 통계 옵트인(기본 꺼짐).

## 권한 (최소·정당화)

| 권한 | 이유 |
|---|---|
| `declarativeNetRequest` | 위협 도메인 접속을 **로컬**에서 차단(방문 URL 서버 전송 없이) |
| `storage` | 위협 목록 캐시·설정·차단 통계 저장 (`local`/`session`) |
| `alarms` | 목록 주기 동기화(~30분), 임시 허용 만료 처리 |
| `contextMenus` | 링크/선택 텍스트 우클릭 검사 |
| `sidePanel` | 상세 근거 패널 |
| `notifications` | 차단 발생 시 알림 |
| `tabs` | 팝업/사이드패널에서 현재 탭 주소 확인 |
| host: `http://localhost:8080/*` | 게이트웨이 목록 동기화 및 명시적 상세검사/신고 |
| content_scripts `http/https` | 호버 툴팁·폼 가드 주입 (모든 웹페이지) |

원격 코드 실행 없음(MV3 준수). 모든 스크립트는 확장에 번들됩니다.

## 게이트웨이 계약 (참조)

- `GET /api/blocklist/snapshot` → `{version, hash, count, entries:[{value, kind, source, severity}]}`
  - `kind:"domain"` 항목만 DNR 차단 대상. `severity`는 `danger`/`warning`.
- `GET /api/check?value=<url>` → `{grade, risk_score, reasons[], organization, feed_sources[], community_reports, recommendation}`
  - "상세검사"·사이드 패널에서만 호출(명시적).
- `POST /api/report` `{target, kind, note}` → 신고 접수.

## 문제 해결

- **차단이 안 될 때:** `chrome://extensions`에서 확장을 새로고침한 뒤,
  팝업 하단 "지금 동기화"로 목록을 받아왔는지(버전 표시) 확인하세요.
- **DNR 규칙 확인:** `chrome://extensions` → 확장 "세부정보" → "서비스 워커"
  콘솔에서 오류를 확인하거나, `chrome://net-internals` 대신 실제 접속으로
  인터스티셜 표시 여부를 확인하세요.
- **상세검사가 오프라인이라고 나올 때:** `localhost:8080` 게이트웨이 실행 여부
  확인. 로컬 판정(호버·팝업 등급)은 게이트웨이 없이도 동작합니다.
- **오탐(정상 사이트 차단):** 인터스티셜의 "오탐 신고" 또는 설정의 개인
  허용목록에 도메인을 추가하세요.
- 코드 수정 후에는 반드시 확장을 새로고침하세요.
